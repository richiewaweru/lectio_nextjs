# Guided Discovery

## Intent
Give learners enough explanation to orient themselves, then confirm the concept through an interactive or visual check.

## Best For
- Concepts that benefit from a short explanation before interaction
- STEM lessons where a simulation should verify the theory, not replace it
- Sections that need sidebar vocabulary support

## Not Ideal For
- Pure exploration lessons that should open with the simulation
- Narrative topics with little or no interactive payoff

## Learner Fit
- General
- Scaffolded
- Analytical

## Subjects
- Mathematics
- Physics
- Biology
- Chemistry

## Lesson Flow
Hook -> Explain -> Define -> Simulate -> Example -> Practice -> Reflect -> What Next

## Required Components
- SectionHeader
- HookHero
- ExplanationBlock
- SimulationBlock
- PracticeStack
- WhatNextBridge

## Optional Components
- DefinitionCard
- WorkedExampleCard
- PitfallAlert
- GlossaryRail
- DiagramBlock
- ReflectionPrompt

## Why This Template Exists
Some lessons need a clear conceptual runway before learners interact. Guided Discovery keeps the explanation legible, then uses the simulation to prove and deepen the idea.

## Responsive Notes
The glossary rail stays sticky beside the main lesson on larger screens and collapses out of the sidebar flow on smaller screens.

## Print Notes
The simulation should collapse to its fallback diagram, glossary support should flatten into the lesson flow, and worked examples should remain fully visible.
